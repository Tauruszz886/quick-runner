# 玩法循环执行文档

## 目标

本玩法循环的核心目标是让玩家通过持续移动获得经验，升级后提高速度，再用更高速度挑战更远关卡与更高收益区域。金币用于升级养成项，养成项反过来提升经验、金币或速度收益，形成长期循环。

核心循环：

```text
移动 / 跑步机 -> 获得经验 -> 升级 -> 速度提升 -> 更快通关/刷经验
通关/金币区 -> 获得金币 -> 升级装备/钱包/轨迹 -> 提升经验或金币效率
达到等级要求 -> 重生 -> 永久倍率提升 -> 重新推进
```

当前导出类：

```ts
ExperienceLevelConfig {
  level: number
  exp: number
```

说明：

- `exp`：当前等级升级所需经验，用于计算。
- `total_exp`：累计经验，用于定位等级。
- `exp_display_k`、`total_display_k`：展示字段，保持字符串，避免飞书千分位格式影响导出。

- 第 10 关不参与这套金币胜利区。
- 普通区基础金币暂定为 `5`。

### 已接入运行时表现

经验、金币和速度 UI 已接入：

- 移动经验每 `0.2s` 采样玩家位置。
- 移动距离只计算 X/Z 横向距离，不计算 Y 轴。
- 经验 UI 显示等级、当前等级经验、经验进度。
- 速度 UI 显示 `速度：数字`。
- 金币 UI 显示 `金币：数字`。
- 经验飘字和金币飘字使用 `Role.show_dynamic_text(...)`，显示在玩家头顶。
- 飘字字号为 `40`。
- 经验飘字持续 `0.8s`，金币飘字持续 `1.4s`。
- 金币区触发复活时，金币数值立即到账，金币飘字延迟到复活后显示。

复活经验保护：

- 死亡回出生点会触发 `PLAYER_DIED_TO_REBIRTH`。
- 经验系统收到该事件后，会对该玩家接下来约 `1.6s` 的移动经验 tick 做抑制。
- 抑制期间只刷新 `lastPosition`，不加经验、不飘经验字。
- 目的：避免死亡点到出生点的横向位置跳变被误算成移动经验。

## 经验与速度设计

### 经验获得入口

所有经验获取都应进入同一个计算入口，避免跑步机、移动、好友、每日、重生各写一套算法。

建议公式：

```text
最终经验 = 基础经验
  * 跑步机倍率
  * 重生经验倍率
  * 装备经验倍率
  * 好友经验倍率
  * 持续游玩倍率
  * 每日连续倍率
```

基础经验来源：

- 玩家实际移动距离
- 跑步机按秒产出
- 后续活动或任务奖励

### 等级计算

等级以 `ExperienceLevelConfig.total_exp` 为主：

```text
当前累计经验 >= 某等级 total_exp -> 达到该等级
升级所需经验 = 当前等级 exp
```

建议运行时维护：

```text
currentExpTotal
currentLevel
currentLevelExp
nextLevelExp
```

### 速度计算

速度不要直接被所有经验倍率相乘，否则很容易失控。建议速度只由等级曲线和少量速度装备影响：

```text
玩家速度 = 等级基础速度 + 运动鞋速度加成 + 其他明确速度加成
```

经验加成只让玩家更快升级，升级再提升速度。

## 养成系统

### 运动鞋

定位：

- 主要提升速度。
- 可少量提升经验，但不建议和轨迹定位重叠太多。

建议字段：

```text
id
quality
level
upgrade_cost
speed_bonus
exp_bonus
```

### 轨迹

定位：

- 主打经验倍率。
- 可作为玩家长期刷经验的主要养成项。

建议字段：

```text
id
quality
level
upgrade_cost
exp_multiplier
```

### 钱包

定位：

- 主打金币倍率。
- 可影响普通胜利区和双倍金币区的最终金币。

建议字段：

```text
id
quality
level
upgrade_cost
coin_multiplier
```

## 加成系统

### 好友加成

规则：

```text
邀请好友后获得 25% 经验加成
```

建议先按布尔状态实现：

```text
hasInviteBonus = true / false
friendExpMultiplier = hasInviteBonus ? 1.25 : 1
```

后续如需要多好友叠加，再扩展成数量型。

### 持续游玩加成

规则：

```text
每玩 1 分钟获得 1% 经验加成
```

建议设置上限，防止挂机无限膨胀。初版可以先用：

```text
每分钟 +0.01
当局或当天上限待定
```

需要记录：

```text
sessionPlaySeconds
continuousExpBonus
```

### 每日连续加成

规则：

```text
每天连续游玩获得 10% 经验加成
```

建议先定义为每日登录/游玩状态：

```text
dailyStreakDays
dailyExpMultiplier = 1 + dailyStreakDays * 0.10
```

是否需要上限待定。

## 重生系统

重生用于长期循环：

```text
达到等级要求 -> 重生 -> 等级/经验重置或部分重置 -> 获得永久倍率
```

重生应影响：

- 经验倍率
- 金币倍率
- 可能解锁更高级跑步机或装备品质

建议字段：

```text
rebirth_count
required_level
exp_multiplier
coin_multiplier
unlock_note
```

需要先确认：

- 重生后等级是否回到 1。
- 金币是否保留。
- 装备是否保留。
- 每次重生等级要求曲线。

## 跑步机系统

跑步机是独立的经验来源，但不应该绕过统一经验公式。

建议逻辑：

```text
玩家进入跑步机触发区
每秒获得 baseExpPerSecond * treadmillMultiplier
最终仍进入 calculateFinalExp(...)
玩家离开触发区后停止
```

跑步机建议分层：

```text
普通跑步机：1x
高级跑步机：2x
稀有跑步机：5x
重生跑步机：10x 或按重生次数解锁
```

需要编辑器 KV：

```text
QRRole=treadmill_exp_zone
QRModule=对应区域或关卡
QRRuntimeName=跑步机名称
QRBaseExpPerSecond=基础每秒经验
QRTreadmillMultiplier=倍率
QRRequiredRebirth=需要重生次数
```

## 金币计算

金币也应统一入口：

```text
单倍金币 = 基础金币
  * 钱包金币倍率
  * 重生金币倍率
  * 其他活动倍率

最终金币 = 单倍金币 * (双倍区 ? 2 : 1)
```

当前已有：

```text
普通胜利区倍率 = 1
双倍胜利区倍率 = 2
基础金币 = 5
```

后续钱包、重生、活动加成应先叠到单倍金币里，再由双倍区做最后的 `* 2`。

## 建议开发阶段

### 阶段 1：经验等级核心

目标：

- 读取 `ExperienceLevelConfig`
- 玩家移动获得经验
- 经验累计后升级
- 等级变化能刷新速度
- UI 显示等级、经验、经验进度

验收：

- 玩家静止不涨经验。
- 玩家移动时稳定涨经验。
- 升级后速度变化可观察。
- 经验跨多级时能连续升级。

### 阶段 2：速度曲线

目标：

- 根据你后续提供的通关速度建议，建立等级到速度的映射。
- 初版可以先用公式，后续再改成数值表。

验收：

- 低等级能完成前几关但不轻松。
- 中等级能明显更快。
- 高等级不会破坏碰撞和机关节奏。

### 阶段 3：金币与装备

目标：

- 普通/双倍金币区接入最终金币公式。
- 钱包影响金币。
- 轨迹影响经验。
- 运动鞋影响速度。

验收：

- 装备升级后加成即时生效。
- 金币扣减和收益正确。
- 双倍金币区仍等于普通区基础金币的 2 倍，再叠钱包/重生。

### 阶段 4：跑步机

目标：

- 根据地图里已有跑步机区域配置触发区。
- 玩家站上跑步机按秒获得经验。
- 跑步机经验进入统一经验公式。

验收：

- 进入触发区开始产出。
- 离开触发区停止产出。
- 多个跑步机倍率不同。
- 不会因为死亡/传送残留产出状态。

### 阶段 5：好友、持续、每日加成

目标：

- 好友加成：25%。
- 持续游玩：每分钟 1%。
- 每日连续：每天 10%。

验收：

- 加成在经验公式中可追踪。
- UI 能显示当前总经验倍率。
- 每日连续逻辑按日期正确刷新。

### 阶段 6：重生

目标：

- 达到等级要求后可重生。
- 重生提高经验/金币倍率。
- 重生数据持久化。

验收：

- 未达等级不能重生。
- 重生后倍率提升。
- 重生后等级/经验/装备/金币处理符合最终规则。

### 阶段 7：存档

目标：

- 保存玩家等级、经验、金币、装备、重生次数、每日状态。
- 进入游戏后恢复。

验收：

- 重进游戏数据不丢。
- 旧存档字段缺失时能兼容默认值。

